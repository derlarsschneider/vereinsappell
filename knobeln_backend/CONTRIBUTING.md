# Contributing to Knobeln Game Backend

Thank you for your interest in contributing to the Knobeln Game Backend! We welcome contributions from the community to help improve this project.

## Getting Started

1. **Fork the repository** on GitHub.
2. **Clone your fork** to your local machine:
   ```bash
   git clone https://github.com/your-username/knobeln-backend.git
   cd knobeln-backend
   ```
3. **Set up the development environment** (see [README.md](README.md) for details).
4. **Create a new branch** for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Development Workflow

1. **Make your changes** to the codebase.
2. **Run tests** to ensure everything works as expected:
   ```bash
   make test
   ```
3. **Lint your code** to maintain code quality:
   ```bash
   make lint
   ```
4. **Format your code** to follow the project's style guide:
   ```bash
   make format
   ```
5. **Commit your changes** with a descriptive commit message:
   ```bash
   git add .
   git commit -m "Add your descriptive commit message here"
   ```
6. **Push your changes** to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
7. **Create a pull request** from your fork to the main repository.

## Code Style

- Follow [PEP 8](https://www.python.org/dev/peps/pep-0008/) for Python code.
- Use [Google Style Docstrings](https://google.github.io/styleguide/pyguide.html#38-comments-and-docstrings) for documentation.
- Keep lines under 100 characters when possible.
- Use meaningful variable and function names.

## Testing

- Write unit tests for new features and bug fixes.
- Ensure all tests pass before submitting a pull request.
- Update tests when changing existing functionality.

## Pull Request Guidelines

- Provide a clear description of the changes in your pull request.
- Reference any related issues or pull requests.
- Ensure your code is well-tested and passes all checks.
- Keep pull requests focused on a single feature or bug fix.
- Be responsive to feedback and be prepared to make changes if requested.

## Reporting Issues

When reporting issues, please include:

- A clear and descriptive title.
- Steps to reproduce the issue.
- Expected vs. actual behavior.
- Any relevant error messages or logs.
- Your environment (OS, Python version, etc.).
